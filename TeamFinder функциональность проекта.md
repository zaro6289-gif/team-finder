# TeamFinder: функциональность проекта

## Базовая функциональность проекта

Вам предстоит выполнить проект **TeamFinder** – это платформа, на которой разработчики, дизайнеры и другие специалисты смогут находить единомышленников для совместной работы над Pet-проектами. Зарегистрированные пользователи смогут публиковать идеи проектов, находить команду на проект и откликаться на опубликованные предложения.

### Как должно работать веб-приложение

В проекте есть следующие страницы:

- главная,
- страница входа,
- страница регистрации,
- страница пользователя,
- редактирование профиля,
- создание и редактирование проекта,
- страница проекта,
- страница с зарегистрированными пользователями,
- страница смены пароля

### Главная страница

Главная страница отображает список актуальных проектов. Каждый проект представлен в виде карточки с краткой информацией:

- автор проекта (аватар и ФИ) — при клике можно перейти на страницу пользователя,
- название — при клике можно перейти на страницу проекта,
- краткое описание,
- дата публикации.

Проекты отсортированы по дате добавления «от новых к старым».

Реализуется **постраничная пагинация** — на одной странице отображаются 12 проектов.

Для авторизованных пользователей на карточке проекта отображается иконка **«Добавить в избранное»**, а также доступна кнопка **«Создать проект»**.

### Страница регистрации

В проекте доступна система регистрации и аутентификации пользователей.

Обязательные поля для пользователя при регистрации:

- имя,
- фамилия,
- адрес электронной почты,
- пароль.

### Страница входа

После регистрации пользователь переадресовывается на страницу входа. Вход в сервис происходит по адресу электронной почты и паролю.

### Страница пользователя

Публичный профиль участника платформы, где отображаются его данные и проекты. Используется как визитка для знакомства с автором или потенциальным членом команды.

1. **Общая информация**
   - Имя и фамилия;
   - Аватар;
   - Краткое описание/о себе;
   - Контактные данные (email, телефон, GitHub).
2. **Навыки (Skills)**
   - Список технологий, которыми владеет пользователь. Владелец профиля может добавлять и удалять навыки.
3. **Активность**
   - Список проектов, созданных пользователем.
4. **Для владельца профиля**
   - Кнопка «Редактировать профиль»;
   - Кнопка «Добавить проект».
   - Иконка «Поделиться профилем». Ссылка не меняется после редактирования профиля.

В большинстве браузеров, например на основе Chromium, ссылка автоматически скопируется в буфер обмена, а в Safari — выведется на экран в правом верхнем углу, откуда её можно скопировать в буфер обмена.

### Редактирование профиля

Страница позволяет пользователю обновить свою личную информацию и навыки. Проекты редактируются отдельно, на других страницах.

1. **Редактирование общей информации**
   - Имя и фамилия;
   - Аватар;
   - Краткое описание/о себе;
   - Контактные данные (email, телефон, GitHub).
2. **Редактирование навыков**
   - Добавление/удаление технологий (навыков) через текстовое поле с автодополнением.
3. **Кнопка «Сохранить»**
   - Изменения вносятся только после подтверждения.

### Страница проекта

Страница проекта — это детальная карточка, в которой отображается вся информация о Pet-проекте, его цели. Пользователи могут просматривать проект, откликаться на участие.

1. **Основная информация**
    - Название проекта;
    - Автор (ссылка на профиль пользователя);
    - Дата публикации;
    - Описание проекта;
    - Текущий статус (открытый, закрытый).
2. **Отклик на проект**
    - Кнопка «Присоединиться к проекту». При клике на кнопку, зарегистрированный пользователь присоединяется к проекту.
3. **Действия (для владельца проекта)**
    - Кнопки: «Редактировать проект», «Завершить проект».
    - Раздел «Участники проекта»: выводится список пользователей, которые присоединились к проекту (аватарка, Фамилия, Имя – при клике можно перейти в профиль)
4. **Дополнительно**
    - Иконка «Поделиться ссылкой» на проект.

    В большинстве браузеров, например на основе Chromium, ссылка автоматически скопируется в буфер обмена, а в Safari — выведется на экран в правом верхнем углу, откуда её можно скопировать в буфер обмена.

### **Создание и редактирование проекта**

Страница создания/редактирования проекта позволяет пользователю опубликовать новый проект или обновить уже существующий. Страница представлена в виде формы, разбитой на логические блоки. При редактировании все поля предварительно заполнены текущими значениями проекта.

1. **Основная информация**
    - Поле ввода названия проекта (обязательное);
    - Текстовое поле «Описание проекта»;
    - Текущий статус (открытый, закрытый)
2. **Управление публикацией**
    - Кнопка «Опубликовать» (делает проект доступным другим пользователям);
    - При редактировании — кнопка «Сохранить изменения».
3. **Валидация и уведомления**
    - Обязательные поля проверяются перед отправкой;
    - После успешного сохранения — перенаправление на страницу проекта.

### Страница с пользователями

Страница показывает всех зарегистрированных пользователей.

Каждый пользователь представлен в виде карточки с краткой информацией:

- Фамилия имя (переход на страницу пользователя),
- аватар,
- кнопка «Посмотреть контакты».

Пользователи отсортированы по дате добавления «от новых к старым».

Реализуется **постраничная пагинация** — на одной странице отображаются 12 профилей.

### Страница изменения пароля

Доступ к этой странице есть только у залогиненных пользователей, через кнопку **Сменить пароль** в выпадающем меню в правом верхнем углу страницы.

### Разграничение прав

В проекте описаны разные уровни доступа пользователей:

- гость (анонимный пользователь):
  - Просмотр проектов на главной;
  - Просмотр страницы отдельного проекта;
  - Просмотр страниц пользователя;
  - Создание аккаунта.
- аутентифицированный (авторизованный) пользователь:
  - Все права анонимных пользователей;
  - Вход в систему под своим логином и паролем;
  - Выход из системы;
  - Создание, редактирование, завершение своих проектов.
- администратор:
  - Все права авторизованных пользователей;
  - Изменение пароля любого пользователя;
  - Создание, блокировка и удаление аккаунтов;
  - Редактирование и удаление любых проектов.

### Вариант #1. Cтраница Избранное + "участники моих проектов"

### Избранное

Добавлять проекты в избранное может только залогиненный пользователь.

Если незалогиненный пользователь попробует добавить проект в избранное, то приложение попросит его зарегистрироваться или войти в свой аккаунт

Добавить проект в избранное можно с главной страницы и со страницы самого проекта, нажав на соответствующую иконку.

Сам список избранного может просмотреть только его владелец.

1. Пользователь отмечает один проект кликом по иконке **Добавить в избранное**.
2. Пользователь переходит на страницу **Избранное** и просматривает свой список избранных проектов.
3. При необходимости пользователь может удалить проект из избранного кликом на соответствующую иконку.

В этом варианте должна быть предусмотрена фильтрация пользователей по одному из 4 заранее заданных критериев:

- “Авторы избранных проектов”
- “Авторы проектов, в которых я участвую”
- “Пользователи, которым нравятся мои проекты”
- “Участники моих проектов”.

### Вариант #2. Навыки, фильтрация участников по навыкам

### Навыки пользователей

На странице пользователя отображается блок «Навыки» со списком тегов. Управление списком доступно данному пользователю, а просмотр — всем.

1. Отображение

- Блок расположен на странице пользователя и показывает все его навыки в виде тегов.
- Для неавторизованных пользователей и участников, не являющихся владельцем страницы, отображается только список без возможности редактирования.

2. Добавление навыка (только владелец страницы)

- Владелец видит кнопку «+ Добавить навык».
- При нажатии появляется поле ввода с автодополнением и список подсказок до 10 совпадений.
- Если точного совпадения нет, можно создать новый навык прямо из интерфейса («Создать „<название>“»).
- Добавление происходит без перезагрузки страницы; новый тег появляется сразу.

3. Удаление навыка (только владелец страницы)

- У каждого тега для владельца доступна иконка удаления (крестик).
- Удаление происходит без перезагрузки страницы; тег исчезает сразу.

4. Технические детали (реализовано)

- UI-логика реализована в `static/js/skills.js` (отложенный поиск, выбор подсказки, создание нового, добавление/удаление тегов, CSRF).

### Фильтрация пользователей по навыкам

На странице списка пользователей доступен визуальный фильтр по навыкам в виде тегов.

1. Отображение фильтра

- Над сеткой пользователей отображается панель «Фильтр по навыкам».
- Список тегов формируется из всех доступных навыков (отсортирован по алфавиту).

2. Применение фильтра

- Клик по тегу применяет фильтр и формирует URL вида `?skill=<Название навыка>`.
- Активный навык подсвечивается, рядом появляется ссылка «Сбросить», ведущая на `/users/list`.
- В списке остаются только пользователи, у которых среди навыков есть выбранный тег (точное совпадение по названию).

3. Технические детали (реализовано)

- Шаблон фильтра и сетки пользователей — `templates/users/participants.html`.

### Критерии приёмки

- На странице пользователя виден блок «Необходимые навыки». Владельцу страницы доступны кнопки добавления и удаления, остальным — только просмотр.
- Автодополнение предлагает существующие навыки; при отсутствии возможна мгновенная регистрация нового навыка.
- Добавление/удаление навыков не требует перезагрузки страницы.
- На странице списка пользователей отображается панель фильтра с тегами всех навыков, активный фильтр подсвечивается.
- Применение фильтра по `?skill=` оставляет только пользователей, содержащих выбранный навык. Доступна ссылка «Сбросить».
